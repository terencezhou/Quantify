# 龙头股追踪

## 目标

在连板接力环境评分（`market_temperature`）和板块热度排名（`industry_temperature`）的基础上，回答最核心的操作问题：

> **明天应该关注哪只股票？以什么方式参与？**

本模块在 15:30 收盘后运行，从当日热门板块中识别龙头股，评估其连板质量和次日操作价值。

---

## 核心理念：短线龙头 ≠ 行业龙头

短线龙头的核心定义只有一个字：**领涨**。

| 特征 | 表现 | 数据映射 |
|------|------|---------|
| **率先启动** | 题材刚冒头时最先涨停 | 封板时间最早 |
| **涨幅最大** | 整个炒作周期中连板最高 | `连板数`、`max_lianban` |
| **辨识度最高** | 提到这个题材第一反应就是这只票 | 连板天数、封板资金 |
| **抗跌性强** | 炸板后能回封，跟风股炸板则一去不回 | 炸板回封、炸板次数 |

**判断龙头不靠预判，靠跟踪。** 市场通过实际交易"选举"出龙头，我们要做的是尽早识别和确认。

---

## 设计思路：三层漏斗

```
┌─────────────────────────────────────────────┐
│  第一层：连板接力环境（market_temperature）      │
│  Score < 20 → 全市场冰点，不操作               │
│  Score 20-50 → 只做最强龙头，极小仓位           │
│  Score > 50 → 积极寻找龙头                     │
└──────────────────────┬──────────────────────┘
                       ▼
┌─────────────────────────────────────────────┐
│  第二层：板块热度排名（industry_temperature）    │
│  只关注热度排名前 N 的行业板块                    │
│  阶段为"启动"或"发酵"的板块优先                  │
│  "高潮"板块的龙头持筹不追，"退潮"板块回避         │
└──────────────────────┬──────────────────────┘
                       ▼
┌─────────────────────────────────────────────┐
│  第三层：龙头评分与操作建议（本模块核心）          │
│  从热门板块中识别龙头，评估次日操作价值           │
└─────────────────────────────────────────────┘
```

## 数据来源

| 数据 | 来源 | 用途 |
|------|------|------|
| `zt_pool` | DataManager.extra | 今日涨停股（代码、名称、行业、连板数、封板资金、首次封板时间、炸板次数） |
| `zt_pool_previous` | DataManager.extra | 昨日涨停股今日表现（代码、涨跌幅、昨日连板数） |

| `SectorHeat` | industry_temperature | 板块阶段、涨停数、龙头信息 |

## 第一步：候选龙头池

从热度排名前 N 个行业（默认 N=8）中提取所有涨停股，作为候选龙头池。

---

## 第二步：五维评分（0-100 分）

### 维度 1：连板高度分（0-30 分）

连板数是龙头身份最直接的证据。

| 连板数 | 得分 |
|--------|------|
| 1 板（首板） | 8 |
| 2 板 | 14 |
| 3 板 | 20 |
| 4 板 | 24 |
| 5 板 | 27 |
| ≥ 6 板 | 30 |

### 维度 2：板块领先度分（0-25 分）

| 条件 | 得分 |
|------|------|
| 连板数 = 板块最高 **且** 唯一最高 | 25 |
| 连板数 = 板块最高 **但** 有并列 | 15 |
| 连板数 < 板块最高 | 0 |

### 维度 3：封板质量分（0-20 分）

**3a. 封板资金排名（0-12 分）**

| 排名 | 得分 |
|------|------|
| 前 3 名 | 12 |
| 前 10 名 | 8 |
| 前 20 名 | 4 |
| 其他 | 0 |

**3b. 炸板回封加分（+8 分）**

`zt_pool` 的 `炸板次数 >= 1` → +8 分。直接使用数据源字段判断，避免用"最低价 < 涨停价"误判（低开拉板不等于炸板）。炸板回封是极强的龙头信号——盘中被砸开但多头能封回来，说明资金坚决。

**3c. 高炸板扣分（0 ~ -6 分）** ⚡ 新增

直接使用 `zt_pool` 中的 `炸板次数` 字段：

| 炸板次数 | 扣分 |
|---------|------|
| 0-1 次 | 0 |
| 2 次 | -3 |
| ≥ 3 次 | -6 |

> **逻辑**：炸板次数越多，盘中分歧越大，说明多空博弈激烈。即使最终封板，次日延续性也大打折扣。3c 与 3b 可同时生效（如 3 次炸板但最终回封：+8 - 6 = +2 净增分）。

最终维度 3 得分 = clamp(3a + 3b + 3c, 0, 20)

### 维度 4：身份确认分（0-15 分）

对连板股和首板股使用不同的确认逻辑：

**连板股（≥ 2 板）：昨日溢价验证**

从 `zt_pool_previous` 取该股今日涨跌幅，衡量市场对其龙头身份的认可度：

| 今日涨跌幅 | 得分 |
|-----------|------|
| ≥ 5% | 15 |
| 3% ~ 5% | 10 |
| 0% ~ 3% | 5 |
| < 0% | 0 |

**首板股（= 1 板）：封板时间 + 板块地位** ⚡ 优化

首板当日不存在"昨日溢价"数据，无法使用连板股的验证逻辑。改为两个子维度：

**4a. 封板时间（0-8 分）**

| 封板时间 | 得分 |
|---------|------|
| 10:00 前 | 8 |
| 10:00 - 11:30 | 5 |
| 下午开盘到 14:30 | 2 |
| 14:30 后 | 0 |

**4b'. 板块地位（0-7 分）** ⚡ 新增（替代原"竞价强度"）

原设计中首板股的 4b "竞价强度"依赖 `zt_pool_previous`，但首板当日不会出现在昨日涨停池中，导致此项永远为 0。现改为评估该首板股所在板块的热度，作为首板质量的间接佐证：

| 条件 | 得分 |
|------|------|
| 板块涨停 ≥ 5 只 **且** 板块阶段为"发酵"或"高潮" | 7 |
| 板块涨停 ≥ 4 只 **且** 板块阶段为"启动"或"发酵" | 5 |
| 板块涨停 ≥ 3 只 | 3 |
| 否则 | 0 |

> **核心思路**：首板股的价值不在于自身历史，而在于它所处板块的爆发力。板块涨停数多、正处于启动/发酵期，说明这个题材有资金共识，首板股有更高概率被市场"选举"为龙头。

首板确认分 = min(15, 4a + 4b')

### 维度 5：梯队支撑分（0-10 分）

| 板块状态 | 得分 |
|---------|------|
| 阶段 = "发酵" 且 `score_ladder ≥ 15` | 10 |
| 阶段 = "启动" 或 "高潮" | 6 |
| 阶段 = "平稳" | 3 |
| 阶段 = "退潮" | 0 |

### 合成公式

$$Score = \text{clamp}(V1 + V2 + V3 + V4 + V5, \; 0, \; 100)$$

## 第三步：龙头分级

| 总分 | 级别 | 含义 |
|------|------|------|
| ≥ 80 | **S 级** | 明确龙头，连板高、资金认可、板块发酵 |
| 65-80 | **A 级** | 准龙头，身份基本确认 |
| 50-65 | **B+ 级** | 有潜力，需次日竞价确认 |
| 40-50 | **B 级** | 候选，身份未完全确认 |
| 30-40 | **C 级** | 弱候选 |
| < 30 | **D 级** | 跟风股，不显示 |

---

## 第四步：生命周期判断

| 连板数 | 板块阶段 | 生命周期 |
|--------|---------|---------|
| 1 板 | 启动 | 确立期 |
| 2-3 板 | 启动/发酵 | 主升期 |
| ≥ 4 板 | 发酵/高潮 | 加速期 |
| 任意 | 退潮 | 衰退期 |

---

## 第五步：操作建议矩阵

将龙头级别与市场阶段交叉，给出明日操作建议：

| 龙头级别 | 冰点期 (0-20) | 混沌/修复期 (20-50) | 主升/发酵期 (50-80) | 高潮期 (80-100) |
|---------|---------------|-------------------|-------------------|-----------------|
| **S** | 轻仓参与（逆势龙） | 半仓参与 | **重仓核心标的** | 持筹享溢价，不追高 |
| **A** | 轻仓跟踪，等确认 | 半仓参与 | 半仓参与 | 持筹，逢高减仓 |
| **B+** | 观望 | 轻仓，关注次日竞价确认 | 半仓参与 | 轻仓试错 |
| **B** | 观望 | 观察不动手 | 轻仓试错 | 观望 |
| **C** | 回避 | 回避 | 观望 | 回避 |

---

## 第六步：风险信号

以下信号触发时，在报告中标注风险提示：

| 信号 | 含义 |
|------|------|
| 板块退潮 | 跟风资金撤退，龙头难以独善其身 |
| 板块高潮 + 连板 ≥ 5 | 关注次日冲高兑现窗口 |
| 连板 ≥ 7 | 高位博弈，非核心参与者回避 |
| 市场高潮期 | 注意高潮后分歧 |
| 市场冰点 (< 20 分) | 极度谨慎 |
| 炸板 ≥ 3 次 | 封板分歧大，次日不确定性高 |
| 炸板 2 次 | 封板有分歧 |

## 输出数据结构

```python
@dataclass
class LeaderStock:
    code: str                   # 股票代码
    name: str                   # 股票名称
    sector: str                 # 所属热门行业
    sector_phase: str           # 板块阶段（启动/发酵/高潮/退潮/平稳）
    lianban: int                # 当前连板数

    score_lianban: float        # 连板高度分 (0-30)
    score_leading: float        # 板块领先度分 (0-25)
    score_seal: float           # 封板质量分 (0-20)
    score_confirm: float        # 身份确认分 (0-15)
    score_support: float        # 梯队支撑分 (0-10)

    score: float                # 总分 (0-100)
    grade: str                  # S / A / B+ / B / C / D
    lifecycle: str              # 确立期 / 主升期 / 加速期 / 衰退期
    action: str                 # 明日操作建议
    risk_flags: List[str]       # 风险信号列表

    seal_amount: float          # 封板资金（元）
    first_seal_time: str        # 首次封板时间
    zhaban_count: int           # 炸板次数
```

---

## 注意事项

1. **时间语义**：脚本在 15:30 后运行，所有评分基于当日收盘数据。"操作建议"指向明日操作。
2. **首板股评分天花板**：首板股在维度 1（连板高度 8 分）和维度 2（板块领先度 0 分）天然较低。维度 4 通过"板块地位"补偿，使得优质首板可获得合理评分（B/B+ 级），避免被系统性低估。
3. **炸板次数**：直接使用 `zt_pool` 数据源的 `炸板次数` 字段，无需推算。高炸板同时影响封板质量分（维度 3 扣分）和风险信号提示。
4. **板块切换**：当一个板块退潮时，资金可能迅速切换到新板块。关注"启动"阶段的新板块，即使其龙头得分暂时较低。
5. **D 级不显示**：报告中只展示 C 级及以上的龙头，D 级（跟风股）自动过滤。

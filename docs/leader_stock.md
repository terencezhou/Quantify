# 寻找龙头股

## 目标

在市场情绪温度计（`market_temperature`）和行业热度分析（`industry_temperature`）的基础上，进一步回答最核心的操作问题：

> **今天应该关注哪只股票？**

本模块的任务是从热门行业中筛选出每个题材的**龙头股**，并评估其当前状态（是否值得参与、以什么方式参与）。

---

## 核心理念：短线龙头 ≠ 行业龙头

基本面语境中的"龙头"是茅台、宁德时代这类市占率最高、盈利最强的公司。短线语境中的龙头完全不同，核心定义只有一个字：**领涨**。

龙头的四大特征：

| 特征 | 表现 | 数据映射 |
|------|------|---------|
| **率先启动** | 题材刚冒头时最先涨停 | 封板时间最早 |
| **涨幅最大** | 整个炒作周期中累计涨幅最高、连板最高 | `max_lianban`、累计涨幅 |
| **辨识度最高** | 提到这个题材，第一反应就是这只票 | 连板天数、市场关注度（封单量） |
| **抗跌性强** | 炸板后能回封，跟风股炸板则一去不回 | 炸板回封率、分时韧性 |

龙头往往"不讲道理"——可能业绩亏损、市值很小、主营关联度不强，但它就是涨得最猛。因为短线交易的本质不是交易公司价值，而是交易**市场情绪和资金共识**。龙头的产生有偶然性：流通盘大小合适、股价位置吸引眼球、被有影响力的资金率先选中。

**判断龙头不靠预判，靠跟踪。** 市场通过实际交易"选举"出龙头，我们要做的是尽早识别和确认。

---

## 设计思路：三层漏斗

```
┌─────────────────────────────────────────────┐
│  第一层：宏观情绪过滤（market_temperature）      │
│  Score < 20 → 全市场冰点，不操作               │
│  Score 20-50 → 只做最强龙头，极小仓位           │
│  Score > 50 → 积极寻找龙头                     │
└──────────────────────┬──────────────────────┘
                       ▼
┌─────────────────────────────────────────────┐
│  第二层：板块筛选（industry_temperature）        │
│  只关注热度排名前 N 的行业板块                    │
│  阶段为"启动"或"发酵"的板块优先                  │
│  "高潮"板块的龙头持筹不追，"退潮"板块回避         │
└──────────────────────┬──────────────────────┘
                       ▼
┌─────────────────────────────────────────────┐
│  第三层：个股龙头识别与评分（本模块核心）          │
│  从筛选出的板块中，找到真正的龙头股               │
│  评估龙头当前状态，给出操作建议                   │
└─────────────────────────────────────────────┘
```

## 已有数据盘点

在动手之前，先明确我们**已经有什么**、**还缺什么**：

### 已有数据（可直接使用）

| 数据 | 来源 | 能提取的龙头信息 |
|------|------|----------------|
| `zt_pool` | DataManager.extra | 今日涨停股全量（代码、名称、行业、连板数、封板资金、涨跌幅、流通市值） |
| `zt_pool_previous` | DataManager.extra | 昨日涨停股今日表现（代码、名称、涨跌幅、昨日连板数） |
| `zt_pool_strong` | DataManager.extra | 强势股池（连续多日强势的股票） |
| `_get_market_df()` | realtime / daily 重建 | 全市场行情（最新价、最高、最低、昨收、涨跌幅、成交量、成交额、换手率） |
| `daily/{code}.parquet` | 本地缓存 | 个股历史 K 线（可追溯连板历史、计算累计涨幅） |
| `SectorHeat.leader_code/name` | industry_temperature | 每个热门行业已识别的"连板最高股" |
| `SectorHeat.lianban_dist` | industry_temperature | 每个行业的连板梯队分布 |
| `SectorHeat.phase` | industry_temperature | 板块所处阶段 |

### 暂时缺少但可以推算的数据

| 数据 | 推算方法 |
|------|---------|
| **封板时间** | `zt_pool` 中的 `最新价==涨停价` 只能判断是否封板，无法得知精确封板时间。可从分时数据（`intraday/{code}.parquet`）推算最早触及涨停价的时间。**但 intraday 数据只有 5 分钟级别**，精度有限。 |
| **封单金额** | `zt_pool` 中的 `封板资金` 字段可直接使用，反映资金锁仓意愿 |
| **炸板回封** | 当前 `zt_pool` 是收盘快照，无法直接判断盘中是否炸板后回封。可间接推断：若该股在 `_get_market_df()` 中 `最低价 < 涨停价` 且最终收于涨停，则盘中可能炸板后回封 |
| **首日竞价强度** | 从 `zt_pool_previous` 中取昨日涨停股，对比今日 `_get_market_df()` 的开盘价与昨收，计算竞价高开幅度 |
| **连板历史轨迹** | 从 `daily/{code}.parquet` 回溯，找到连续涨停的起始日和路径 |

## 第一步：候选龙头池构建

### 1.1 从热门行业提取种子

输入：`industry_temperature` 输出的 `List[SectorHeat]`（按热度排序）

取热度排名前 N 个行业（建议 N=5~8），从每个行业的 `zt_pool` 涨停股中提取候选：

```
候选池 = 所有热门行业中今日涨停的股票（来自 zt_pool）
```

### 1.2 初步过滤

剔除以下股票：
- ST / *ST 股
- 上市不足 60 天的次新股（连续一字板不算真龙头）
- 流通市值过大（> 500 亿）的股票：游资拉不动，短线弹性差
- 流通市值过小（< 10 亿）的股票：容量不够，大资金进出困难

---

## 第二步：龙头评分（0-100 分）

对候选池中每只股票，从五个维度进行量化评分：

### 维度 1：连板高度分（0-30 分）

连板数是龙头身份最直接的证据。连板越高，市场共识越强。

| 连板数 | 得分 |
|--------|------|
| 1 板（首板） | 5 |
| 2 板 | 12 |
| 3 板 | 18 |
| 4 板 | 22 |
| 5 板 | 26 |
| ≥ 6 板 | 30 |

> **为什么首板只有 5 分**：首板阶段龙头身份尚未确认，大量首板中只有极少数能晋级为真龙头。2 板是龙头身份的第一道确认关卡。

### 维度 2：板块领先度分（0-25 分）

龙头必须是板块内最强的那只。衡量它相对于同板块其他涨停股的领先程度。

| 条件 | 得分 |
|------|------|
| 该股连板数 = 板块最高连板 **且** 唯一最高 | 25 |
| 该股连板数 = 板块最高连板 **但** 非唯一（有并列） | 15 |
| 该股连板数 < 板块最高连板（不是最强的） | 0 |

> **关键逻辑**：同一板块内连板最高且唯一最高的那只才有"龙头溢价"。如果有两只都是 3 板，说明龙头尚未分化，需要等下一天的竞价来确认。

### 维度 3：封板质量分（0-20 分）

反映资金对该股的认可度和锁仓决心。

**3a. 封板资金排名（0-12 分）**

将候选池按 `封板资金` 降序排名：

| 排名 | 得分 |
|------|------|
| 前 3 名 | 12 |
| 前 10 名 | 8 |
| 前 20 名 | 4 |
| 其他 | 0 |

**3b. 炸板回封加分（0-8 分）**

如果该股盘中最低价 < 涨停价（说明盘中炸过板），但收盘仍封板 → **加 8 分**。

> 炸板回封是极强的龙头信号——说明有分歧但多头能力更强，"炸不开"本身就是实力的体现。反之，一字板虽然看起来强，但缺少分歧换手，后续可能加速见顶。

### 维度 4：昨日溢价分（0-15 分）

如果该股昨天也涨停（即今天是连板而非首板），看它今天给昨天打板者带来了多少溢价。

从 `zt_pool_previous` 找到该股（如果存在），取其 `涨跌幅`：

| 今日涨跌幅 | 得分 |
|-----------|------|
| ≥ 5%（含涨停一字） | 15 |
| 3% ~ 5% | 10 |
| 0% ~ 3% | 5 |
| < 0%（昨天打板亏钱） | 0 |

> **首板股不适用此维度**：首板股在 `zt_pool_previous` 中不存在，此项默认 0 分。这也是首板股天然得分低的原因之一——龙头身份需要至少一天的市场验证。

### 维度 5：梯队支撑分（0-10 分）

龙头不能是"孤军奋战"，必须有跟风梯队支撑。这个分数直接复用所在板块的 `SectorHeat` 数据。

| 板块状态 | 得分 |
|---------|------|
| 板块阶段 = "发酵" 且 `score_ladder ≥ 15` | 10 |
| 板块阶段 = "启动" 或 "高潮" | 6 |
| 板块阶段 = "平稳" | 3 |
| 板块阶段 = "退潮" | 0 |

> **退潮板块的龙头不做**：即便它还在涨停，板块退潮意味着跟风资金撤退，龙头很快会成为"独角戏"。

## 第三步：龙头状态判断与操作建议

计算出总分后，还需要结合板块阶段和市场情绪给出操作层面的建议。

### 3.1 龙头分级

| 总分 | 级别 | 含义 |
|------|------|------|
| ≥ 80 | **S 级龙头** | 明确的板块龙头，连板高、资金认可、板块发酵中 |
| 60-80 | **A 级龙头** | 准龙头，身份基本确认，值得重点跟踪 |
| 40-60 | **B 级候选** | 潜在龙头，身份未完全确认（可能是首板或板块刚启动） |
| < 40 | **跟风股** | 不构成龙头特征，回避 |

### 3.2 操作建议矩阵

将龙头级别与市场情绪、板块阶段交叉，给出具体操作建议：

| 龙头级别 | 市场冰点期 (0-20) | 混沌期 (20-50) | 主升期 (50-80) | 高潮期 (80-100) |
|---------|-------------------|----------------|----------------|-----------------|
| **S 级** | 小仓试错（冰点反弹博弈） | 半仓参与 | **重仓核心标的** | 持筹享溢价，不追高 |
| **A 级** | 观望 | 轻仓跟踪 | 半仓参与 | 持筹，准备兑现 |
| **B 级** | 观望 | 观望 | 轻仓试错 | 回避 |
| **跟风** | 回避 | 回避 | 回避 | 回避 |

### 3.3 关键风险信号

以下信号出现时，无论龙头评分多高，都应该**提示风险**：

| 信号 | 含义 | 处理 |
|------|------|------|
| 龙头所在板块阶段从"高潮"转"退潮" | 板块见顶，龙头最后的溢价 | 提示"次日冲高兑现" |
| 市场情绪从高潮期掉至主升期以下 | 全局退潮，龙头也难独善其身 | 提示"控制仓位" |
| 龙头连板 ≥ 7 且 全市场涨停数骤降 | 空间龙见顶信号 | 提示"高位博弈，非核心参与者回避" |
| 昨日溢价 R_yu < 0 | 打板者集体亏钱 | 提示"接力链断裂，观望一天" |

## 第四步：早期龙头识别（首板阶段辅助判断）

在题材第一天爆发时（板块阶段 = "启动"），通常有多只首板股同时涨停，此时龙头尚未分化。以下线索帮助在首板阶段做初步预判：

### 4.1 封板时间优先

> **谁先涨停，谁更可能成为龙头。**

从分时数据（`intraday/{code}.parquet`，5 分钟级别）中找到最早触及涨停价的时间。在同板块首板股中，封板时间最早的 1-2 只标记为"领先封板"。

| 封板时间 | 加分 |
|---------|------|
| 上午 10:00 前 | +5 |
| 10:00-11:30 | +3 |
| 下午开盘后 | +1 |
| 尾盘封板（14:30 后） | 0 |

### 4.2 次日竞价确认

**龙头的确认往往发生在第二天竞价阶段。** 这是最关键的一步：

从 `_get_market_df()` 取该股今日开盘价和昨收（来自 `zt_pool_previous` 的代码在今日行情中匹配）：

```
竞价强度 = (今日开盘价 - 昨日收盘价) / 昨日收盘价
```

| 竞价强度 | 信号 |
|---------|------|
| ≥ 5%（含一字板） | 极强确认，龙头身份几乎确定 |
| 3% ~ 5% | 强确认 |
| 0% ~ 3% | 正常分化 |
| < 0%（低开） | 弱于预期，暂不确认龙头身份 |

在同板块昨日首板股中，**竞价强度最高的 1 只**大概率是龙头。

### 4.3 流通盘适配

游资偏好流通市值 **30-150 亿** 的标的：
- 太小（< 15 亿）：容量不够，大资金进出困难，容易被操纵
- 太大（> 300 亿）：拉升需要的资金量太大，短线弹性差
- 适中（30-150 亿）：既有足够的弹性，又有足够的容量

| 流通市值 | 适配分 |
|---------|--------|
| 30-150 亿 | 最佳 |
| 15-30 亿 或 150-300 亿 | 可接受 |
| < 15 亿 或 > 300 亿 | 不利 |

## 关键设计原则

### 1. 龙头是"选举"出来的，不是预判的

系统的核心逻辑不是"预测谁会成为龙头"，而是**尽早确认市场已经选出的龙头**。所有评分维度都基于已经发生的市场行为（连板数、封板资金、竞价表现），而非基本面预判。

### 2. 做龙头不做跟风

龙头的容错率远高于跟风股。龙头炸板后可能回封，跟风股炸板则直接跌停。系统应该明确标注"跟风股"并建议回避，即使它看起来"便宜"。

### 3. 龙头的高度不是风险，龙头的走弱才是风险

连板 5 板、6 板的龙头看起来"太高"，但只要它还在被资金追捧（封单厚、板块发酵中），它就是市场上最安全的短线标的。真正的风险信号是：龙头开始封不住板、板块开始退潮、市场整体情绪转冷。

### 4. 与现有模块的关系

```
market_temperature（全局情绪）
    └── industry_temperature（板块热度）
            └── leader_stock（龙头识别）← 本模块
```

本模块**依赖**前两个模块的输出，不重复计算。市场情绪决定仓位上限，板块热度决定关注方向，龙头识别决定具体标的。

## 注意事项

1. **数据时效性**：龙头评分基于收盘后复盘数据，盘中实时变化无法反映。次日竞价前需要重新评估。
2. **封板时间精度**：分时数据为 5 分钟级别，封板时间精度有限。如果两只股票在同一个 5 分钟窗口内封板，无法进一步区分。
3. **一字板龙头的特殊性**：连续一字板（如 3 个一字板）的龙头缺少换手，后续一旦开板可能剧烈波动。对连续一字板 ≥ 3 天的龙头应标注"一字板风险"。
4. **板块切换**：当一个板块退潮时，资金可能迅速切换到新板块。系统应关注"启动"阶段的新板块，即使其龙头得分暂时较低。
5. **市值数据来源**：`zt_pool` 中的 `流通市值` 字段可直接使用；若缺失，可从 `_get_market_df()` 或 `stock_info` 缓存补充。

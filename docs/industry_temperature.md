# 板块热度排名

## 目标

在连板接力环境评分（`market_temperature`）给出全局评分之后，进一步拆解到**行业板块**维度，回答：

1. 当前哪些行业是热点？
2. 每个热点行业有多"热"？（量化热度排名）
3. 每个热点行业处于什么阶段？（启动 / 发酵 / 高潮 / 退潮）

---

## 数据来源

| 数据 | 来源 | 关键字段 |
|------|------|---------|
| `zt_pool` | DataManager.extra | `代码, 名称, 所属行业, 连板数, 涨跌幅, 封板资金` |
| `zt_pool_previous` | DataManager.extra | `代码, 名称, 所属行业, 昨日连板数, 涨跌幅` |
| `stock_sector_map.parquet` | 本地缓存 | `代码 → 所属行业`（从 zt_pool 家族累积） |
| `sector_flow` | DataManager.extra | `名称, 今日主力净流入-净额, 今日主力净流入-净占比` — 行业主力资金流向 |
| `daily/{code}.parquet` | 本地日K缓存 | `日期, 成交额` — 计算成交额放量比（回退方案） |
| `_get_market_df(trade_date)` | realtime / daily 重建 | `代码, 名称, 最新价, 最高, 昨收, 涨跌幅` |

> **资金数据策略**：优先使用 `sector_flow`（东财 `stock_sector_fund_flow_rank`，通过付费代理稳定可用），可直接获取行业主力净流入金额和占比。若 `sector_flow` 不可用，回退到日K成交额放量比。

---

## 计算步骤

### Step 1：按行业聚合涨停数据

从 `zt_pool` 按 `所属行业` 分组，统计每个行业的：

| 指标 | 计算方式 |
|------|---------|
| `zt_count` | 该行业涨停股数量 |
| `max_lianban` | 该行业最高连板数（龙头高度） |
| `lianban_dist` | 连板数分布，如 `{1: 5, 2: 2, 3: 1}` |
| `leader_code` / `leader_name` | 最高连板股的代码和名称 |
| `total_seal_amount` | 封板资金合计（反映资金力度） |

**过滤**：剔除 ST，剔除 `zt_count < 2` 的行业（单只涨停不构成板块效应）。

### Step 2：计算连板梯队完整度

对每个行业的 `lianban_dist` 评估梯队质量：

```
梯队层级 = max_lianban（最高连板数）
梯队宽度 = 有几个不同的连板级别（1板、2板、3板…）
```

评分规则：

| 条件 | 梯队得分（0~25） |
|------|-----------------|
| 仅有首板 | 5 |
| 有 1板 + 2板 | 12 |
| 有 1板 + 2板 + 3板（三级梯队） | 20 |
| 四级及以上梯队 | 25 |

### Step 3：计算资金活跃度得分（主力净流入排名 / 成交额放量比）

**优先方案：使用 `sector_flow`（东财行业资金流排行）**

`sector_flow` 提供每个行业板块的 `主力净流入-净额` 和 `主力净流入-净占比`，按净流入金额降序排名：

| 排名 | 资金活跃度得分（0~15） |
|------|---------------------|
| 前 5 名 | 15 |
| 前 10 名 | 12 |
| 前 20 名 | 8 |
| 前 50% | 4 |
| 后 50% | 0 |

**回退方案：`sector_flow` 不可用时，使用日K成交额放量比**

从 `daily/{code}.parquet` 并发读取所有个股当日成交额及近5日均成交额，按 `stock_sector_map` 分行业聚合：

```
板块放量比 = 板块当日成交额合计 / 板块近5日均成交额合计
```

同样按放量比全市场排名，评分规则相同。

> **主力净流入 vs 放量比**：主力净流入能直接反映大资金的方向（流入还是流出），更精准。放量比衡量"异常活跃度"但无法区分方向。两者互补，优先用前者。

### Step 4：计算跟风广度

利用 `stock_sector_map.parquet` 找到该行业的所有已知成分股，再从 `_get_market_df()` 中查询这些股票当日涨跌幅：

```
跟风率 = 行业成分股中涨幅 > 3% 的股票数 / 行业总成分股数
```

| 跟风率 | 跟风得分（0~10） |
|--------|-----------------|
| ≥ 30% | 10 |
| 10%~30% | 线性插值 |
| ≤ 10% 或成分股不足 5 只 | 0 |

> 若 `stock_sector_map` 中某行业成分股不足 5 只，跳过此维度，默认 0 分。

### Step 5：计算涨停集中度得分

```
集中度 = 该行业涨停数 / 全市场涨停数（zt_pool 总数）
```

| 集中度 | 集中度得分（0~30） |
|--------|-------------------|
| ≥ 15% | 30 |
| 3%~15% | 线性插值 |
| ≤ 3% | 0 |

### Step 6：合成行业热度总分

```
热度 = 涨停集中度得分(30) + 梯队完整度得分(25) + 龙头高度得分(20) + 资金活跃度得分(15) + 跟风广度得分(10)
```

龙头高度得分规则：

| 最高连板 | 龙头得分（0~20） |
|----------|-----------------|
| ≥ 5 板 | 20 |
| 4 板 | 16 |
| 3 板 | 12 |
| 2 板 | 8 |
| 1 板 | 4 |

总分范围 **0~100 分**。

### Step 7：判断板块阶段

比较今日（`zt_pool`）与昨日（`zt_pool_previous`）该行业的涨停数和龙头高度变化：

| 今日 vs 昨日 | 阶段 | 标志 |
|-------------|------|------|
| 昨日该行业涨停 ≤ 1，今日 ≥ 3 | **启动** | 新题材异军突起 |
| 连续 2 天涨停 ≥ 3，龙头在晋级（今日连板 > 昨日连板） | **发酵** | 龙头确认，梯队跟进 |
| 龙头已到 ≥ 5 板，跟风股大面积涨停，跟风率高 | **高潮** | 全面开花，谨慎追高 |
| 龙头断板（今日连板 < 昨日），或涨停数骤减 ≥ 50% | **退潮** | 资金撤退，回避 |
| 不满足以上条件 | **平稳** | — |

> 昨日该行业涨停数 = `zt_pool_previous` 中该行业（按 `所属行业` 聚合）的股票数。注意 `zt_pool_previous` 的含义是"昨日涨停股在今日的表现"，其 `所属行业` 可用于聚合昨日涨停的行业分布。

## 输出格式

### 数据结构

```python
@dataclass
class SectorHeat:
    sector: str           # 行业名称
    zt_count: int         # 涨停数
    max_lianban: int      # 最高连板
    leader_code: str      # 龙头代码
    leader_name: str      # 龙头名称
    lianban_dist: dict    # 连板分布 {1: 5, 2: 2, 3: 1}
    total_seal_amount: float  # 封板资金合计（元）
    sector_turnover: float    # 板块成交额（元）
    net_inflow: float         # 主力净流入（元），来自 sector_flow
    net_inflow_pct: float     # 主力净流入占比（%）
    follow_rate: float    # 跟风率
    score: float          # 热度总分（0~100）
    phase: str            # 阶段：启动/发酵/高潮/退潮/平稳
    fund_source: str      # 资金数据来源: sector_flow / daily_volume
```

## 注意事项

1. **行业名截断**：`zt_pool.所属行业` 的行业名经常被截断（如"汽车零部"应为"汽车零部件"），与 `stock_sector_map` 的行业名一致（均来源于 zt_pool 家族），内部匹配无问题。
2. **非交易日**：与连板接力环境评分一致，非交易日使用最近交易日缓存。
3. **冷门行业**：涨停数 < 2 的行业不参与排名，避免噪声。
4. **`stock_sector_map` 覆盖不足**：该映射只从 zt_pool 家族积累，只包含近期涨停过的股票，对从未涨停的冷门股无覆盖。跟风广度指标在成分股 < 5 只时不计分。
5. **资金活跃度**：优先使用东财 `sector_flow` 的主力净流入排名（通过付费代理稳定可用），回退到成交额放量比（当日/近5日均值）。报告输出中同时展示"主力净流入"和"成交额"两列。

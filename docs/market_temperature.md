# 打板复盘报告 — 连板接力环境评分

## 目标

基于五大核心数据维度，计算 **0-100 分**的连板接力环境评分，判断当前市场处于何种打板阶段，并给出明日操作策略建议。

> **核心定位**：本评分专注衡量"打板接力"环境的好坏——即市场当前是否适合做连板接力、首板打板等短线操作。脚本在 15:30 收盘后运行，评分代表"今日已发生的接力环境"，用于推断明日操作的激进/保守程度。

---

## 数据来源

| 数据 | 来源 | 用途 |
|------|------|------|
| `zt_pool` | DataManager.extra（`stock_zt_pool_em`） | 涨停数 U、炸板数 B、最高连板 H、连板分布 |
| `zt_pool_previous` | DataManager.extra（`stock_zt_pool_previous_em`） | 昨日涨停溢价 R_yu、连板晋级率 PR |
| 全市场行情 | 优先级递减：1. `extra['realtime']` 2. `realtime_quotes.parquet` 3. `daily/` K线并发重建 | 跌停数 D、大面数 M |
| 交易日历 | `ak.tool_trade_date_hist_sina()` → 本地缓存 `trade_calendar.parquet`（90天有效期） | 确定基准交易日 |

> 非交易时段（周末/节假日）自动回退到最近交易日数据。

## 第一步：提取五大核心数据维度

### 维度 1：涨跌停数据

| 指标 | 定义 | 数据来源 |
|------|------|---------|
| **U** — 有效涨停家数 | 剔除 ST、未开板新股 | `zt_pool` 行数 |
| **D** — 跌停家数 | 剔除 ST，最新价 ≤ 跌停价（昨收 × (1 - 涨跌停幅度)） | `_get_market_df()` 全市场行情 |
| **M** — 大面数量 | 当日最高→收盘回撤 ≥ 10% 的非ST股 | `_get_market_df()` 全市场行情 |
| **B** — 炸板家数 | 最高 ≥ 涨停价 且 收盘 < 涨停价（排除已封板的 `zt_pool` 股票） | `_get_market_df()` 全市场行情 |

> **涨跌停幅度按板块区分**：主板(60/00) ±10%、创业板(300/301) ±20%、科创板(688) ±20%、北交所(430/830/870) ±30%。

### 维度 2：封板成功率

$$FR = \frac{U}{U + B}$$

- FR > 75% → 打板安全，情绪高涨
- FR < 60% → 打板危险，情绪恶劣

### 维度 3：昨日涨停今日溢价率

$$R_{yu} = \text{昨日涨停股今日收盘平均涨跌幅}$$

- R_yu > 2% → 赚钱效应极好，资金继续猛冲
- R_yu < 0% → 核按钮，接力意愿冰点

计算方式：取 `zt_pool_previous` 中昨日涨停股的 `涨跌幅` 列均值。若 `zt_pool_previous` 缺少涨跌幅，从 `_get_market_df()` 按代码匹配补充。

### 维度 4：连板晋级率

$$PR = \frac{\text{今日连板家数（连板数} \ge 2\text{）}}{\text{昨日首板及以上家数}}$$

- 今日连板家数：从 `zt_pool` 中筛选 `连板数 ≥ 2` 的股票数
- 昨日首板及以上家数：`zt_pool_previous` 的总行数（`昨日连板数` 列 ≥ 1）

### 维度 5：市场最高空间

$$H = \max(\text{zt\_pool 中的连板数})$$

- H ≤ 3 → 冰点退潮
- H ≥ 7 → 情绪主升浪

---

## 第二步：构建评分公式

满分 100 分，权重分配如下：

### 1. 赚钱效应得分（满分 40 分）

| R_yu | 得分 |
|------|------|
| ≥ 3% | 40 |
| ≤ -2% | 0 |
| 中间 | 线性插值：`(R_yu + 2) / 5 × 40` |

### 2. 打板安全度得分（满分 20 分）

| FR | 得分 |
|----|------|
| ≥ 80% | 20 |
| ≤ 50% | 0 |
| 中间 | 线性插值：`(FR - 0.50) / 0.30 × 20` |

### 3. 接力意愿得分（满分 20 分）

| PR | 得分 |
|----|------|
| ≥ 30% | 20 |
| ≤ 10% | 0 |
| 中间 | 线性插值：`(PR - 0.10) / 0.20 × 20` |

### 4. 恐慌扣分项（-20 ~ 0 分）

$$Panic = D + 0.5 \times M$$

| Panic | 扣分 |
|-------|------|
| ≥ 30 | -20 |
| ≤ 5 | 0 |
| 中间 | 线性插值 |

### 5. 空间溢价加分项（0 ~ 20 分，阶梯式平滑）

| 条件 | 得分 |
|------|------|
| H ≥ 6 **且** U ≥ 60 | +20 |
| H ≥ 5 **且** U ≥ 50 | +10 |
| H ≥ 4 **且** U ≥ 40 | +5 |
| 否则 | 0 |

### 合成公式

$$Score = Score_{赚钱} + Score_{安全} + Score_{接力} + Score_{恐慌} + Score_{空间}$$

Clamp 到 [0, 100]。

## 第三步：明日操作策略

| 分数 | 阶段 | 数据特征 | 明日操作策略 |
|------|------|---------|---------|
| 0-20 | **冰点期** | 跌停成片(D>20)，R_yu<0，H≤3板 | 空仓。连续两天<15分可小仓位试错新题材首板 |
| 20-50 | **混沌/修复期** | 跌停减少，R_yu回暖至~1%，晋级率低 | 半仓，只做前排核心，不做后排跟风 |
| 50-80 | **主升/发酵期** | U>60，FR>75%，连板梯队完整(1板到6板) | 重仓出击！积极参与龙头接力和核心题材2进3 |
| 80-100 | **高潮期** | 几乎无跌停，R_yu>4%，大量一字板 | 持筹享溢价，停止买入，准备冲高兑现，防分歧下杀 |

## 第四步：明日策略倾向

在操作策略之外，进一步输出**策略排序建议**，帮助判断"明天应该用什么方式操作"。

### 情绪方向判断

完全基于原始数据（`zt_pool` + `zt_pool_previous`）推导，不缓存任何派生分数：

- `U_prev` = `len(zt_pool_previous)`（剔除 ST 后）= 昨日涨停数
- `H_prev` = `zt_pool_previous['昨日连板数'].max()` = 昨日最高连板

判断规则（按优先级）：

| 优先级 | 条件 | 方向 | 含义 |
|--------|------|------|------|
| 1 | H_prev >= 5 且 H < H_prev - 1 | **下降** | 空间龙断板，最强退化信号 |
| 2 | R_yu < 0 且 U < U_prev | **下降** | 赚钱效应转负 + 涨停缩量 |
| 3 | R_yu > 2 且 U >= U_prev | **上升** | 赚钱效应强 + 涨停不缩 |
| 4 | H > H_prev 且 H >= 5 | **上升** | 新高度打开 |
| 5 | 其余 | **震荡** | — |

### 策略排序规则

| 条件组合 | 策略排序 | 理由 |
|---------|---------|------|
| 冰点期 | 观望 > 冰点试错 | 空仓为主，连续冰点可试错首板 |
| 方向=下降 | 观望 > 低吸（极轻仓） | 情绪退化中，等企稳再动手 |
| 方向=上升 + PR>=20% | 接力 > 半路 > 低吸 | 接力环境回暖，龙头有溢价 |
| 方向=上升 + PR<20% | 半路 > 低吸 > 接力 | 赚钱效应好但晋级率低 |
| 方向=震荡 + PR>=20% | 接力 > 趋势 > 轮动 | 有连板基础 |
| 方向=震荡 + FR>=70% | 半路 > 低吸 > 轮动 | 封板率尚可 |
| 方向=震荡 + 有启动板块 | 轮动 > 半路 > 趋势 | 新板块启动中 |
| 方向=震荡 + 其余 | 趋势 > 轮动 > 观望 | 无明确方向 |

> "有启动板块" = `hot_sectors` 中存在 `phase == '启动'` 的板块。

---

## 第五步（Step 6）：板块热度排名

在连板接力环境评分计算完成后，自动调用 `IndustryTemperature` 进行行业维度拆解：

- 识别涨停 ≥ 2 只的热点行业
- 量化每个行业热度（0-100 分）
- 判断板块阶段（启动/发酵/高潮/退潮/平稳）
- 结果存入 `TemperatureResult.hot_sectors: List[SectorHeat]`

详见 [industry_temperature.md](industry_temperature.md)。

## 输出数据结构

```python
@dataclass
class TemperatureMetrics:
    U: int = 0          # 有效涨停家数
    D: int = 0          # 跌停家数
    M: int = 0          # 大面数量
    B: int = 0          # 炸板家数
    FR: float = 0.0     # 封板率
    R_yu: float = 0.0   # 昨日涨停溢价率（%）
    PR: float = 0.0     # 连板晋级率
    H: int = 0          # 最高连板数

@dataclass
class TemperatureResult:
    metrics: TemperatureMetrics
    score_earn: float       # 赚钱效应得分 (0-40)
    score_safety: float     # 打板安全度 (0-20)
    score_relay: float      # 接力意愿 (0-20)
    score_panic: float      # 恐慌扣分 (-20~0)
    score_space: float      # 空间溢价 (0-20)
    score: float            # 连板接力环境评分 (0-100)
    phase: str              # 冰点期/混沌/修复期/主升/发酵期/高潮期
    strategy: str           # 明日操作策略
    trend: str              # 情绪方向：上升/震荡/下降
    strategy_hint: str      # 策略排序，如 "低吸 > 半路 > 接力"
    strategy_reason: str    # 一句话理由
    trade_date: str         # 基准交易日 (YYYYMMDD)
    hot_sectors: list       # List[SectorHeat] 热门行业
```

## 注意事项

1. **交易日判断**：使用 `ak.tool_trade_date_hist_sina()` 获取 A 股真实交易日历，本地缓存 90 天。回退到简单的"周一到周五"逻辑作为兜底。
2. **全市场行情降级**：当 `realtime` 数据不可用（非交易时段/缺失），自动从 `daily/*.parquet` 并发重建（约 4000+ 文件，~8 秒），带内存缓存避免重复读取。
3. **炸板数 B**：直接从 `_get_market_df()` 计算（最高 ≥ 涨停价 且 收盘 < 涨停价），排除 `zt_pool` 已封板代码防双计。涨停价按板块区分幅度。
4. **大面判定**：`(最高 - 最新价) / 最高 ≥ 0.10`，剔除 ST 和停牌（最高=0），不限制涨跌幅方向。
5. **核心指标 R_yu**：是最重要的单一指标（权重 40/100），直接衡量"昨天追板的人今天赚不赚钱"，是资金接力意愿的核心驱动。
6. **时间语义**：脚本在 15:30 后运行，所有指标反映的是"今日已发生"的市场状态。评分结果用于推断明日操作的激进/保守程度——分数代表"今日接力环境的惯性"，分数越高明日可操作性越强。

# 均线回归策略 — 多头排列首日捕捉

## 一、策略核心理念

### 1.1 什么是均线回归

均线回归是指短周期均线从分散、交织状态重新收拢并形成**多头排列**的过程。当 MA5、MA10、MA20、MA30 依次从上到下排列，说明短中期资金方向一致向上，趋势确立。

```
典型均线回归过程：

  均线缠绕/交叉  →  短线均线上穿中线均线  →  完全多头排列
  （方向不明）       （趋势萌芽）            （趋势确立）
```

### 1.2 策略目标

捕捉均线回归的**第一天**——从"不是多头排列"变成"多头排列"的临界时刻。这一天通常意味着：

- 短线资金方向与中线资金方向刚刚达成一致
- 趋势启动初期，空间尚未打开，风险收益比最佳
- 是介入的最佳时机窗口（比均线完全发散后再追要安全得多）

### 1.3 核心逻辑

> **均线从缠绕到排列的那一刻，是多空力量从平衡转向多头优势的拐点。第一天出现回归意味着趋势刚刚确立，市场尚未充分反应。**

## 二、均线回归定义

### 2.1 多头排列条件

在交易日 T，同时满足以下三个条件即视为**均线回归状态**：

#### 条件 1：均线多头排列

```
MA5_T > MA10_T > MA20_T > MA30_T

其中：
  MA5_T  = 日期T的5日收盘价简单移动平均
  MA10_T = 日期T的10日收盘价简单移动平均
  MA20_T = 日期T的20日收盘价简单移动平均
  MA30_T = 日期T的30日收盘价简单移动平均
```

含义：短线均线在最上方，中线均线在最下方，四条均线像楼梯一样从上到下排列，是典型的多头趋势结构。

#### 条件 2：MA5 斜率向上

```
slope_ma5 = (MA5_T - MA5_(T-1)) / MA5_(T-1)

要求: slope_ma5 > 0

即: MA5 今天的值 > MA5 昨天的值
```

含义：仅有多头排列不够，MA5 必须仍在走高，说明最近 5 天的平均买入成本在持续提升，短线资金仍在积极进场。如果 MA5 走平或拐头，即便排列正确也可能是趋势末期。

#### 条件 3：MA10 斜率向上

```
slope_ma10 = (MA10_T - MA10_(T-1)) / MA10_(T-1)

要求: slope_ma10 > 0

即: MA10 今天的值 > MA10 昨天的值
```

含义：MA10 代表两周平均成本。MA10 也向上说明不只是最短线的 5 日资金在推动，两周维度的资金也在持续流入。MA5 + MA10 双斜率向上，能过滤掉仅靠 1-2 天大阳线硬拉出排列的假回归。

### 2.2 用公式表示

```
is_alignment(T) = (MA5_T > MA10_T > MA20_T > MA30_T)
                  AND (MA5_T > MA5_(T-1))
                  AND (MA10_T > MA10_(T-1))
```

### 2.3 为什么选 5/10/20/30 四条均线

| 均线 | 代表含义 | 选用原因 |
|------|---------|---------|
| MA5 | 一周平均成本 | 最灵敏的短线风向标 |
| MA10 | 两周平均成本 | 短线与中线的桥梁 |
| MA20 | 一月平均成本 | 最核心的中期趋势线 |
| MA30 | 一个半月平均成本 | 中期趋势的确认线 |

四条均线覆盖了 1 周到 1.5 个月的维度，足以确认短中期趋势方向一致，又不会因为引入 MA60/MA120 等长期均线而错过短线机会。

## 三、首日回归判定

### 3.1 首日回归条件

均线回归的价值在于**变化的那一刻**，而不是持续处于多头排列的每一天。

```
is_first_alignment(T) = is_alignment(T) AND NOT is_alignment(T-1)

即：
  T 日满足均线回归条件
  T-1 日不满足均线回归条件（任一子条件不成立即可）
```

### 3.2 T-1 不满足的可能情形

T-1 日不是均线回归，意味着以下至少一个条件成立：

| 情形 | 说明 | 含义 |
|------|------|------|
| MA5 < MA10 | 5日线在10日线下方 | 短线仍弱于中短线 |
| MA10 < MA20 | 10日线在20日线下方 | 中短线仍弱于中线 |
| MA20 < MA30 | 20日线在30日线下方 | 中期趋势尚未翻多 |
| MA5 走平或向下 | slope_ma5 <= 0 | 短线动能不足 |
| MA10 走平或向下 | slope_ma10 <= 0 | 中短线动能不足 |

只要 T 日这些全部修正过来，就完成了从"非多头"到"多头"的跨越。

### 3.3 边界情况处理

| 场景 | 处理 |
|------|------|
| T-1 是多头排列但 MA5 斜率 <= 0，T 日 MA5 斜率 > 0 | 视为首日回归（斜率从非正转正） |
| T-1 的 MA5 = MA10（相等而非大于） | T-1 不满足严格大于，若 T 满足则算首日 |
| 连续多天均线回归后中断 1 天又恢复 | 再次恢复的那天算新一轮首日回归 |
| 数据不足（新股上市不足 30 天） | 无法计算 MA30，跳过不判定 |

## 四、回归状态评分

### 4.1 评分逻辑

均线回归的质量取决于各条均线的方向。四条均线（MA5、MA10、MA20、MA30）各自向上则说明对应周期的资金成本在抬升，趋势越健康。

#### 评分规则：按均线向上个数打分，满分 4 分

```
score = 0

if MA5_T  > MA5_(T-1):   score += 1    # 5日均线向上
if MA10_T > MA10_(T-1):  score += 1    # 10日均线向上
if MA20_T > MA20_(T-1):  score += 1    # 20日均线向上
if MA30_T > MA30_(T-1):  score += 1    # 30日均线向上
```

### 4.2 分数含义

| 分数 | 级别 | 均线方向 | 含义 | 操作建议 |
|------|------|---------|------|---------|
| 4 | 极强 | 5/10/20/30 全部向上 | 全周期共振，趋势刚确立且各周期动能充足 | 最佳介入时机，可正常仓位买入 |
| 3 | 强 | 3 条向上，1 条走平或微下 | 多数周期共振，通常是 MA30 尚在拐头过程 | 可介入，留意落后均线的方向变化 |
| 2 | 中 | 2 条向上，2 条走平或向下 | 短线有动能但中线支撑不足 | 观望为主，等待更多均线转向 |
| 1 | 弱 | 仅 MA5 向上 | 仅短线有反弹迹象 | 不操作，回归质量不佳 |

### 4.3 评分示例

```
示例 1（4 分 — 完美回归）：
  MA5:  12.50 → 12.68 (↑)  +1
  MA10: 12.30 → 12.35 (↑)  +1
  MA20: 12.10 → 12.15 (↑)  +1
  MA30: 11.90 → 11.92 (↑)  +1
  排列: 12.68 > 12.35 > 12.15 > 11.92 ✓
  MA5斜率: (12.68-12.50)/12.50 = +1.44% ✓
  → 首日回归且评分 4 分：全周期共振的理想信号

示例 2（3 分 — 强回归）：
  MA5:  15.80 → 16.05 (↑)  +1
  MA10: 15.50 → 15.60 (↑)  +1
  MA20: 15.20 → 15.25 (↑)  +1
  MA30: 15.00 → 14.98 (↓)  +0
  排列: 16.05 > 15.60 > 15.25 > 14.98 ✓
  MA5斜率: (16.05-15.80)/15.80 = +1.58% ✓
  → 首日回归且评分 3 分：MA30 尚在拐头，整体趋势向好

示例 3（排列满足但仅 1 分 — 弱回归）：
  MA5:  20.10 → 20.15 (↑)  +1
  MA10: 20.00 → 19.98 (↓)  +0
  MA20: 19.80 → 19.75 (↓)  +0
  MA30: 19.50 → 19.45 (↓)  +0
  排列: 20.15 > 19.98 > 19.75 > 19.45 ✓
  MA5斜率: +0.25% ✓
  → 首日回归但评分仅 1 分：仅 MA5 向上，中线均在下行，不建议介入
```

## 五、实现逻辑（伪代码）

### 5.1 核心判定函数

```python
def check_ma_alignment(df, target_date):
    """
    判断指定日期是否出现均线回归首日。

    参数:
        df: 包含日K数据的DataFrame，至少需要 'close' 列和日期索引
        target_date: 待检测的目标日期 (str 或 date，格式 'YYYY-MM-DD')

    返回:
        dict: {
            'date': 目标日期,
            'is_alignment': bool — 是否处于均线回归状态,
            'is_first_day': bool — 是否为首日回归,
            'score': int (0-4) — 回归质量评分,
            'detail': dict — 各均线值与方向明细,
        }
    """
    ma_periods = [5, 10, 20, 30]

    for p in ma_periods:
        df[f'MA{p}'] = df['close'].rolling(p).mean()

    t = get_row(df, target_date)
    t_prev = get_row(df, prev_trading_day(target_date))

    # --- 条件1: 多头排列 ---
    is_bull_order = (t['MA5'] > t['MA10'] > t['MA20'] > t['MA30'])

    # --- 条件2: MA5 斜率向上 ---
    ma5_slope_up = (t['MA5'] > t_prev['MA5'])

    # --- 当日是否处于回归状态 ---
    is_alignment = is_bull_order and ma5_slope_up

    # --- 前一日是否处于回归状态 ---
    prev_bull_order = (t_prev['MA5'] > t_prev['MA10'] > t_prev['MA20'] > t_prev['MA30'])
    t_prev2 = get_row(df, prev_trading_day(prev_trading_day(target_date)))
    prev_ma5_slope_up = (t_prev['MA5'] > t_prev2['MA5'])
    prev_is_alignment = prev_bull_order and prev_ma5_slope_up

    # --- 首日判定 ---
    is_first_day = is_alignment and (not prev_is_alignment)

    # --- 评分 ---
    score = 0
    directions = {}
    for p in ma_periods:
        up = t[f'MA{p}'] > t_prev[f'MA{p}']
        if up:
            score += 1
        directions[f'MA{p}'] = 'up' if up else 'down'

    return {
        'date': target_date,
        'is_alignment': is_alignment,
        'is_first_day': is_first_day,
        'score': score,
        'detail': {
            'ma_values': {f'MA{p}': round(t[f'MA{p}'], 2) for p in ma_periods},
            'ma_directions': directions,
            'ma5_slope': round((t['MA5'] - t_prev['MA5']) / t_prev['MA5'] * 100, 4),
        }
    }
```

### 5.2 灵活日期输入

支持多种日期输入格式，并进行合法性校验：

```python
def parse_input_date(date_str):
    """
    解析用户输入的日期字符串。

    支持格式:
        '2025-03-10', '20250310', '2025/03/10'

    校验:
        1. 格式是否合法
        2. 是否为交易日（非周末、非节假日）
        3. 是否有足够历史数据（至少前推 30+2 个交易日）

    返回:
        date: 解析后的日期对象
    """
    formats = ['%Y-%m-%d', '%Y%m%d', '%Y/%m/%d']
    parsed = None
    for fmt in formats:
        try:
            parsed = datetime.strptime(date_str.strip(), fmt).date()
            break
        except ValueError:
            continue

    if parsed is None:
        raise ValueError(f"无法解析日期: {date_str}，支持格式: YYYY-MM-DD / YYYYMMDD / YYYY/MM/DD")

    if not is_trading_day(parsed):
        nearest = find_nearest_trading_day(parsed)
        warn(f"{parsed} 非交易日，自动调整至最近交易日 {nearest}")
        parsed = nearest

    return parsed
```

### 5.3 批量扫描（全市场扫描某一日期的首日回归）

```python
def scan_first_alignment(all_stocks_data, target_date):
    """
    全市场扫描：在 target_date 这天，哪些股票首次出现均线回归。

    参数:
        all_stocks_data: {(code, name): daily_df, ...}
        target_date: 目标日期

    返回:
        list[dict]: 满足首日回归的股票列表，按 score 降序排列
    """
    results = []
    for (code, name), df in all_stocks_data.items():
        if len(df) < 35:
            continue
        result = check_ma_alignment(df, target_date)
        if result['is_first_day']:
            result['code'] = code
            result['name'] = name
            results.append(result)

    results.sort(key=lambda x: x['score'], reverse=True)
    return results
```

### 5.4 区间扫描（回测某只股票在时间段内的所有首日回归）

```python
def backtest_alignment(df, start_date, end_date):
    """
    回测：找出某只股票在指定时间段内所有的首日回归事件。

    返回:
        list[dict]: 每次首日回归的日期、评分、后续N日涨跌幅
    """
    events = []
    trading_days = get_trading_days_between(start_date, end_date)

    for date in trading_days:
        result = check_ma_alignment(df, date)
        if result['is_first_day']:
            future_returns = calc_future_returns(df, date, periods=[3, 5, 10, 20])
            result['future_returns'] = future_returns
            events.append(result)

    return events
```

## 七、进阶扩展

### 7.1 辅助过滤条件

均线回归是纯技术面信号，实战中建议叠加以下过滤：

| 过滤条件 | 规则 | 原因 |
|----------|------|------|
| 量能配合 | 回归日成交量 > MA5量 | 有量支撑的回归更可靠 |
| 收盘价位置 | 收盘价 > MA5 | 价格在短线均线之上，强势特征 |
| 非高位 | 距 120 日低点涨幅 < 50% | 排除高位的均线粘合后排列 |
| 排除 ST | 名称不含 ST / *ST | 退市风险 |
| 排除北交所 | 代码不以 8/4 开头 | 流动性和涨跌幅机制不同 |
| 排除科创、创业板| - | 我无法买 |
| 流通市值 | 30 亿 ~ 500 亿 | 排除极端小盘和超大盘 |

### 7.2 追高风险过滤

均线回归首日不等于安全买点。同样是首日回归，均线缠绕后缓缓张开（安全）和一根大阳线硬拉出排列（追高）完全不同。

#### 三个追高度量指标

| 指标 | 计算方式 | 含义 |
|------|---------|------|
| 价格偏离MA5 (BIAS5) | `(close - MA5) / MA5` | 收盘价超出5日均线的幅度。理想值 0~2%，>3% 偏高，>5% 明显追高 |
| 均线扩散度 | `(MA5 - MA30) / MA30` | MA5和MA30的距离。首日回归理想情况是刚刚分开（<3%），>5% 中风险，>8% 高风险 |
| 近5日累计涨幅 | `(close_T - close_(T-5)) / close_(T-5)` | 直接衡量近期涨幅。<5% 安全，>8% 中风险，>13% 高风险 |

#### 综合风险判定

```
风险等级 = f(BIAS5, 扩散度, 5日涨幅)

🟢 低风险: 三个指标全部低于"中"阈值
🟡 中风险: 恰好一个指标达到"中"阈值
🔴 高风险: 任一指标达到"高"阈值，或两个及以上达到"中"阈值
```

#### 阈值配置

| 指标 | 中风险阈值 | 高风险阈值 |
|------|-----------|-----------|
| BIAS5 | ≥ 3% | ≥ 5% |
| 均线扩散度 | ≥ 5% | ≥ 8% |
| 5日累计涨幅 | ≥ 8% | ≥ 13% |

#### 操作建议

| 追高风险 | 建议 |
|----------|------|
| 🟢 低 | 正常操作，均线缓慢回归，风险可控 |
| 🟡 中 | 轻仓参与，严格止损 |
| 🔴 高 | 不追，等回踩MA5或MA10后再观察 |

### 7.3 MA5 斜率的量化细分

基础版只要求 MA5 斜率 > 0，进阶版可对斜率大小做分级：

```
slope_ma5 = (MA5_T - MA5_(T-1)) / MA5_(T-1) × 100%

斜率分级:
  > 1.0%  — 急升型回归（短线拉升强劲，但需警惕回踩）
  0.3%~1.0% — 稳健型回归（最理想，上升有力且不过激）
  0.1%~0.3% — 温和型回归（慢牛启动特征，适合中线持有）
  0%~0.1%  — 微弱型回归（勉强向上，需关注是否会再次拐头）
```

### 7.3 回归持续性跟踪

首日回归只是入口信号，后续需要跟踪回归状态是否持续：

```
首日回归（T日）→ 买入/关注
  ├─ T+1~T+3：均线排列持续 → 回归有效，持股/加仓
  ├─ T+1~T+3：MA5 拐头向下 → 回归失败，考虑止损
  └─ T+5 后：MA5 下穿 MA10 → 排列被破坏，趋势结束
```

## 九、策略局限与注意事项

### 9.1 局限性

| 局限 | 说明 | 应对 |
|------|------|------|
| 滞后性 | 均线本身是滞后指标，回归信号出现时股价已有一定涨幅 | 结合量价配合和K线形态进一步确认 |
| 假回归 | 均线短暂排列后又交叉回去（尤其在震荡市） | 用评分过滤：仅关注 3-4 分的高质量回归 |
| 熊市失效 | 下跌趋势中偶尔的反弹也能形成短暂排列 | 结合 market_temperature，冰点期不操作 |
| 对横盘股无效 | 长期横盘的股票均线缠绕频繁触发 | 叠加量能和涨幅位置过滤 |

### 9.2 最佳适用环境

| 市场环境 | 回归信号可信度 | 建议 |
|----------|--------------|------|
| 牛市 / 主升期 | 高 | 正常使用，回归后持续上涨概率大 |
| 震荡市 | 中 | 仅关注 4 分信号，快进快出 |
| 熊市 / 下跌期 | 低 | 不使用，绝大多数是假回归 |

### 9.3 与均线金叉的区别

| 对比项 | 均线金叉 | 均线回归（本策略） |
|--------|---------|-----------------|
| 判定条件 | MA5 上穿 MA10 | MA5 > MA10 > MA20 > MA30 四线排列 |
| 要求均线条数 | 2 条 | 4 条 |
| 斜率要求 | 无 | MA5 必须向上 |
| 信号强度 | 单一交叉信号较弱 | 四线共振信号更强 |
| 适用周期 | 短线 | 短中线 |

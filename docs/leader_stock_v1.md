# 龙头股识别 v1 — 五层漏斗 + K线形态增强

## 目标

在 `leader_stock.md` 原有"三层漏斗 + 五维度打分"的基础上，新增**K线形态确认**层和**筹码确认**层，形成完整的五层漏斗。重点引入两种高置信度的启动前K线形态：**横盘整理后突破**和**底部圆弧底**。

最终输出按总分降序排列的龙头股列表，每只股票附带逐维度的打分明细和理由。

## 第一层：宏观情绪过滤

直接复用 `market_temperature` 输出的 `score`（0-100）。

| 市场阶段 | score 区间 | 动作 |
|----------|-----------|------|
| 冰点 | < 20 | **终止流程**，不寻找龙头 |
| 混沌/修复 | 20~50 | 继续，但后续仓位建议打五折 |
| 主升/发酵 | 50~80 | 正常流程 |
| 高潮 | > 80 | 继续，但标注"持筹不追高" |

**输出**：`market_phase`（阶段名）、`position_cap`（仓位上限系数 0~1）

---

## 第二层：板块筛选

直接复用 `industry_temperature` 输出的 `List[SectorHeat]`。

1. 取热度排名前 N 个行业（N = 5~8）
2. 从每个行业的 `zt_pool` 涨停股中提取候选
3. 过滤规则：
   - 剔除 ST / *ST
   - 剔除上市不足 60 天的次新股
   - 剔除流通市值 > 500 亿 或 < 10 亿
   - 板块阶段 = "退潮" 的整体剔除

**输出**：候选池 `List[(code, name, sector, sector_phase)]`，约 50~100 只

## 第三层：龙头身份评分（0-100 分）

沿用原 `leader_stock.md` 的五维度打分，对候选池逐只评分。

### 维度 A：连板高度分（0-30）

| 连板数 | 得分 | 理由模板 |
|--------|------|---------|
| 1 板（首板） | 5 | `首板，龙头身份未确认` |
| 2 板 | 12 | `2连板，通过首道晋级关卡` |
| 3 板 | 18 | `3连板，市场辨识度建立` |
| 4 板 | 22 | `4连板，板块空间龙` |
| 5 板 | 26 | `5连板，强势龙头` |
| >= 6 板 | 30 | `{N}连板，绝对龙头` |

### 维度 B：板块领先度分（0-25）

| 条件 | 得分 | 理由模板 |
|------|------|---------|
| 连板数 = 板块最高 且 唯一最高 | 25 | `板块唯一最高板，龙头地位明确` |
| 连板数 = 板块最高 但有并列 | 15 | `与{X}并列最高板，龙头待分化` |
| 连板数 < 板块最高 | 0 | `板块内非最高板（最高: {X} {N}板）` |

### 维度 C：封板质量分（0-20）

**C1 封板资金排名（0-12）**：候选池内按封板资金降序排名

| 排名 | 得分 | 理由模板 |
|------|------|---------|
| 前 3 名 | 12 | `封板资金Top3（{amount}亿）` |
| 前 10 名 | 8 | `封板资金Top10（{amount}亿）` |
| 前 20 名 | 4 | `封板资金Top20（{amount}亿）` |
| 其他 | 0 | `封板资金排名靠后（{amount}亿）` |

**C2 炸板回封（0-8）**：基于分时数据检测"封板→砸开→回封"

> 注意：不能简单用"日K最低价 < 涨停价"判断，因为几乎所有非一字板涨停都满足该条件。
> 必须通过分时数据确认股价曾经到达涨停、离开涨停、再次回到涨停。

```
检测逻辑（使用 intraday 5分钟K线）:

  0. 计算涨停价:
     zt_price = round(昨收 * 1.10, 2)           # 主板/创业板 10%
     zt_price = round(昨收 * 1.20, 2)           # 科创板/北交所 20%
     tolerance = 0.015                           # 容差 1.5 分钱

  1. 找首次封板 bar:
     遍历分时 bar，找第一个 abs(bar.收盘 - zt_price) <= tolerance 的 bar
     → 记为 T_seal（首次封板时间）
     若未找到 → 非涨停股，跳过

  2. 找炸板 bar:
     从 T_seal 之后遍历，找第一个 bar.收盘 < zt_price - tolerance 的 bar
     → 记为 T_break（炸板时间）
     若未找到 → 一路封住，未炸板

  3. 确认回封:
     最后一根 bar（收盘 bar）的 abs(收盘 - zt_price) <= tolerance
     → 收盘封住

  三步都满足 → 炸板回封 ✅
```

**关于数值精度**：
- 涨停价计算时 `round(昨收 * 1.10, 2)` 可能与交易所实际涨停价存在 ±0.01 元的误差（交易所用四舍五入到分）
- 分时数据的价格也是精确到分（0.01 元）
- 因此比较时使用 `tolerance = 0.015`（1.5 分钱），覆盖浮点误差和四舍五入偏差，同时不会误匹配非涨停价格（涨停附近的价差通常 >= 0.03 元）

| 检测结果 | 得分 | 理由模板 |
|----------|------|---------|
| 炸板后回封 | 8 | `盘中{T_seal}封板→{T_break}炸板→回封，多头控盘力强` |
| 一路封住未炸板 | 0 | — |
| 一字板（全天封住） | 0 | — |
| 无分时数据 | 0 | 无法判断，不加分 |

### 维度 D：昨日溢价分（0-15）

仅适用于连板股（首板默认 0 分）。

| 今日涨跌幅 | 得分 | 理由模板 |
|-----------|------|---------|
| >= 5% | 15 | `昨日打板今日获利{pct}%，接力意愿强` |
| 3%~5% | 10 | `昨日打板今日获利{pct}%，正常溢价` |
| 0%~3% | 5 | `昨日打板今日微利{pct}%，溢价偏弱` |
| < 0% | 0 | `昨日打板今日亏损{pct}%，接力链断裂` |

### 维度 E：梯队支撑分（0-10）

| 板块状态 | 得分 | 理由模板 |
|---------|------|---------|
| 阶段=发酵 且 梯队充足 | 10 | `板块发酵中，梯队{N}只跟风涨停` |
| 阶段=启动 或 高潮 | 6 | `板块{phase}期，梯队尚可` |
| 阶段=平稳 | 3 | `板块平稳，梯队薄弱` |
| 阶段=退潮 | 0 | `板块退潮，梯队崩塌` |

### 龙头身份分汇总

```
身份分 = A + B + C + D + E    (满分 100)
```

| 身份分 | 级别 | 处理 |
|--------|------|------|
| >= 80 | S 级 | 进入第四层 |
| 60~79 | A 级 | 进入第四层 |
| 40~59 | B 级 | 进入第四层 |
| < 40 | 跟风股 | **剔除**，不进入第四层 |

## 第四层：K线形态确认（v1 新增核心）

对第三层输出的每只候选龙头（grade >= B），读取其历史日K线数据，从两个子维度计算**形态加分**（-20 ~ +30）。

### 子维度 F1：启动前K线形态（-10 ~ +20）

逐只检测以下形态（互斥，命中第一个即停止）：

#### 形态 ①：横盘整理后突破（+20）

```
检测逻辑:
  1. 取涨停前 N 日（N = 20~60）的日K
  2. 计算该窗口内的振幅: (最高价 - 最低价) / 最低价
  3. 计算该窗口内的换手率均值

判定条件（同时满足）:
  - 窗口振幅 < 15%（窄幅震荡）
  - 窗口天数 >= 20（至少横盘 20 个交易日）
  - 涨停当日突破窗口最高价
  - 涨停当日量比 > 1.5（放量确认）

评分: +20
理由模板: `横盘{N}日(振幅{amp}%)后放量突破，蓄力充分`
```

**为什么给最高分**：横盘时间越长，筹码换手越充分，突破后上方几乎没有套牢盘，拉升阻力最小。这是短线龙头启动的最理想形态。

#### 形态 ②：底部圆弧底（+15）

```
检测逻辑:
  1. 取涨停前 40~90 日的日K
  2. 将窗口等分为三段: 前段、中段、后段
  3. 分别计算三段的收盘价均值: avg_1, avg_2, avg_3

判定条件（同时满足）:
  - avg_1 > avg_2（前段高于中段 → 前期下跌）
  - avg_3 > avg_2（后段高于中段 → 后期回升）
  - avg_2 为三段最低（中段是弧底）
  - 后段的 MA5 > MA20（短期均线已上穿长期 → 趋势反转中）
  - 涨停当日在后段末端

评分: +15
理由模板: `圆弧底形态({N}日)，趋势反转确认后涨停启动`
```

**为什么给次高分**：圆弧底说明股价经历了完整的"下跌→筑底→回升"过程，底部形态越圆润，主力吸筹越充分。但相比横盘突破，圆弧底的突破点不如横盘那样清晰，给 +15。

#### 其他形态（不额外加分）

| 形态 | 分数 | 理由模板 |
|------|------|---------|
| 上升途中加速 | 0 | `上升趋势中加速涨停，无额外形态加分` |
| 急跌后V型反弹 | 0 | `超跌反弹涨停，持续性存疑` |
| 高位横盘后再突破 | -5 | `高位盘整后涨停，警惕最后一冲` |
| 无明显形态 | 0 | `无可识别的启动前形态` |

### 子维度 F2：位置安全度（-10 ~ +10）

```
指标计算:
  rise_from_low = (涨停价 - 60日最低价) / 60日最低价
  dist_to_high  = (120日最高价 - 涨停价) / 涨停价
  ma_arrange    = MA5/MA10/MA20/MA60 的排列关系
```

| 条件 | 分数 | 理由模板 |
|------|------|---------|
| rise_from_low < 15% 且 多头排列 | +10 | `低位启动(距60日低点{pct}%)，均线多头排列` |
| rise_from_low < 30% 且 MA20向上 | +5 | `中低位启动(距60日低点{pct}%)，趋势向上` |
| rise_from_low 30%~60% | 0 | `中位启动(距60日低点{pct}%)，注意前高压力` |
| rise_from_low 60%~100% | -5 | `偏高位启动(距60日低点{pct}%)，追高风险` |
| rise_from_low > 100% 或 空头排列 | -10 | `高位/已翻倍(距60日低点{pct}%)，风险极高` |

### 形态加分汇总

```
形态加分 = F1 + F2    (范围: -20 ~ +30)
```

## 第五层：筹码确认（可选增强）

当 `fetch_chips = true` 且该股有筹码缓存时，额外计算筹码加分（-5 ~ +10）。

### 子维度 G：筹码结构分（-5 ~ +10）

| 条件 | 分数 | 理由模板 |
|------|------|---------|
| 70集中度 < 4% 且 获利比例从低位拉升 | +10 | `筹码单峰密集(集中度{pct}%)，启动确认` |
| 70集中度 < 6% | +5 | `筹码较集中(集中度{pct}%)` |
| 70集中度 6%~10% | 0 | `筹码一般(集中度{pct}%)` |
| 70集中度 > 10% | -5 | `筹码发散(集中度{pct}%)，分歧大，警惕一日游` |

无筹码数据时，G = 0，不影响评分。

## 总分计算与排行

```
总分 = 身份分(A+B+C+D+E) + 形态加分(F1+F2) + 筹码加分(G)
范围: 0 ~ 140（理论值），实际优秀龙头约 70~120
```

### 最终分级

| 总分 | 级别 | 操作建议 |
|------|------|---------|
| >= 100 | **S+** | 最强龙头 + 最佳形态，核心标的，可重仓 |
| 80~99 | **S** | 明确龙头，正常仓位参与 |
| 60~79 | **A** | 准龙头，半仓或轻仓参与 |
| 40~59 | **B** | 潜在龙头，轻仓试探或仅观察 |
| < 40 | — | 已在第三层被剔除 |

最终按总分降序排列输出。

## 硬性否决规则

无论总分多高，出现以下情况直接标红并建议不操作：

| 规则 | 条件 | 处理 |
|------|------|------|
| 板块退潮 | 所在板块 phase = "退潮" | 否决（第二层已剔除，此为兜底） |
| 高位翻倍 + 筹码发散 | rise_from_low > 100% 且 70集中度 > 10% | 否决，标注"高位出货形态" |
| 连续一字板 >= 3 天 | 最近 3 日均为一字涨停（开盘即涨停） | 标注"一字板风险，缺少换手" |
| 昨日溢价为负 + 市场冰点 | D = 0 且 market_score < 20 | 否决，"接力链断裂 + 系统性风险" |

## 与现有模块的关系

```
market_temperature（全局情绪）          → 第一层
    └── industry_temperature（板块热度） → 第二层
            └── leader_stock 五维度打分  → 第三层（沿用）
                    └── K线形态确认      → 第四层（v1 新增）
                            └── 筹码确认 → 第五层（v1 新增，可选）
```
